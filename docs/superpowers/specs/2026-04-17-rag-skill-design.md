# RAG Skill Design Spec

## 개요

HwiCortex 컬렉션을 활용한 범용 RAG 스킬. 사용자가 `/rag`로 호출하면 컬렉션에서 관련 문서를 검색하고, 요청에 따라 문서 작성(write), 평가(review), 또는 검색 결과 제시(search)를 수행한다.

게임기획, 기술문서, 사내규정, 법률 등 도메인에 무관하게 동작한다.

## 요구사항 요약

| 항목 | 결정 |
|------|------|
| 트리거 | 수동만 (`/rag`) |
| 모드 | 자동 판단 기본, 명시 가능 (write/review/search) |
| 컬렉션 지정 | 자유 형태 파싱, 없으면 전체 검색 |
| 출처 명시 | 필수 |
| 검색 횟수 | 제한 없음 (에이전트 판단) |
| 대상 사용자 | 본인 + 외부 사용자 |

## SKILL.md 프론트매터

```yaml
---
name: rag
description: Search hwicortex collections and use results for document writing, evaluation, or search result presentation. Manual trigger only via /rag.
user_invocable: true
---
```

## 호출 인터페이스

```
/rag "전투 시스템 기획서 작성해줘, game-method 컬렉션"
/rag write "보상 루프 설계, game-method"
/rag review ~/projects/my-game/combat.md -c game-method
/rag search "밸런싱 DPS 곡선" -c game-method
```

### 파싱 규칙

1. **모드 감지**: 첫 토큰이 `write`/`review`/`search`이면 해당 모드. 아니면 자동 판단.
   - "작성/만들어줘/기획서/문서 작성" 또는 "write/create/draft" → write
   - "평가/검토/리뷰/분석" 또는 "review/evaluate/assess" → review
   - 그 외 → search
2. **컬렉션 감지**: `-c <name>` 플래그 우선 (확실). 없으면 텍스트에서 컬렉션명 파싱 (best-effort: `hwicortex collection list` 결과와 대조하여 매칭). 둘 다 없으면 전체 컬렉션 검색.
3. **컬렉션 검증**: `hwicortex collection list`로 존재 확인. 없으면 유사 이름 제안.

## 공통 검색 파이프라인

모든 모드가 공유하는 핵심 절차.

### Step 1 — 요청 분석

사용자 요청에서 핵심 주제/키워드를 추출한다. 복합 주제면 하위 주제로 분해한다.

예: "전투 시스템 기획서" → 전투 루프, 밸런싱, 스킬 시스템, 보상

### Step 2 — 컬렉션 확인

```bash
hwicortex collection list
```

파싱된 컬렉션이 목록에 있는지 확인. 없으면 사용자에게 알리고 유사한 컬렉션명을 제안.

### Step 3 — 반복 검색

주제별로 하이브리드 검색을 수행한다.

```bash
# 기본: 하이브리드 검색 (최고 품질)
hwicortex query "<주제>" -c <collection> --full --json -n 5

# 모호한 키워드는 --intent로 맥락 좁히기
hwicortex query "밸런싱" -c <collection> --intent "전투 수치 DPS" --full --json -n 5
```

- 주제마다 검색 수행
- 결과가 부족하면 키워드 변형하여 재검색
- 중복 제거, 관련도 높은 문서 선별

**검색 명령어 선택 기준:**

| 상황 | 명령어 |
|------|--------|
| 기본 (최고 품질) | `hwicortex query` |
| LLM 없이 빠르게 | `hwicortex search` |
| 의미적 유사도만 | `hwicortex vsearch` |
| 특정 문서 전문 조회 | `hwicortex get` / `multi-get` |

1차로 `query`를 사용하되, 보완이 필요하면 `search`나 `get`으로 추가 조회.

### Step 4 — 컨텍스트 구성

선별된 문서들을 정리하여 이후 모드별 작업의 근거로 사용한다.

### 토큰 예산 가이드라인

- 검색 결과 총량 목표: **20,000토큰 이내** (약 80,000자)
- 주제당 `-n 5` 기본, 결과가 충분하면 줄여도 됨
- `--full`은 핵심 문서에만 사용. 탐색 단계에서는 스니펫(기본 출력)으로 관련도 확인 후 필요한 문서만 `hwicortex get`으로 전문 조회
- 단일 문서가 5,000토큰을 초과하면 관련 섹션만 발췌하여 사용

## 모드별 가이드라인

### write 모드

1. 공통 검색 파이프라인 실행
2. 검색된 문서를 기반으로 구조(목차) 초안 제시
3. 사용자 확인 후 섹션별 작성 (사용자가 목차를 수정하면 반영, 특정 섹션 생략 요청도 수용)
4. 각 섹션에 근거가 된 문서를 참조 표기
5. 검색 결과에 없는 내용은 `> 참조 없음 — 일반 지식 기반` 표기

### review 모드

1. 평가 대상 파일을 읽음 (경로 지정 또는 대화 내 텍스트)
2. 대상에서 핵심 주제를 추출하여 공통 검색 파이프라인 실행
3. 검색된 기준 문서에서 평가 축(항목)을 도출 — 기준 문서에 명시된 원칙/규칙이 평가 항목이 됨
4. 항목별로 기준 문서와 대조 평가
5. 결과 포맷:
   - 항목별 평가: 점수(A~F 또는 100점 척도) + 참조 문서 + 근거
   - 기준 문서가 없는 영역은 "해당 기준 없음 — 평가 생략" 표기
   - 개선 제안 (참조 문서의 구체적 섹션 포함)
   - 종합 평가

### search 모드 (기본)

1. 공통 검색 파이프라인 실행
2. 검색 결과를 요약하여 제시 (문서 경로, 관련 내용 스니펫, 관련도)
3. 사용자가 특정 문서를 더 보고 싶으면 `hwicortex get`으로 전문 조회

## 출처 명시 규칙 (전 모드 공통)

```markdown
> 참조: <컬렉션명>/<문서경로>
```

- 모든 근거 있는 주장/내용에 반드시 참조 표기
- 여러 문서를 참조하면 복수 표기
- 검색 결과에 없는 내용은 `> 참조 없음 — 일반 지식 기반` 명시

## 스킬 파일 구조

```
skills/
└── rag/
    └── SKILL.md
```

기존 스킬 패턴과 동일하게 SKILL.md 단일 파일로 구성.

## 에러 처리

- `hwicortex` 명령 실패 시: 에러 메시지 출력 후 가능한 범위에서 작업 계속
- 컬렉션 없음: 사용자에게 알리고 등록 방법 안내
- 검색 결과 없음: "관련 문서를 찾지 못했습니다" 출력 후, 일반 지식 기반으로 작업할지 사용자에게 확인
